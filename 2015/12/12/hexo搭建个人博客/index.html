出去浪了一天，比较累，回来准备早点睡的，但是一想到这一觉一睡又不知道到什么时候了，所以还是先干点事吧。 前些日子没怎么写代码，净瞎整博客的事了，就把这个过程总结下吧。
大概从这个学期初有了给自己搭个博客的想法，主要是我自从大二开始就没怎么发过qq动态了，其实有时候还是很想写点想法以及总结的，但是好友太多，有些话并不想让很多人看到。所以就想着自己申请个域名弄个属于自己的空间，写写各种总结什么的，不为别的，就供自己当个笔记以及今后回顾大学生活的工具。
有想过在当前比较流行的一些社区里注册博客，但是仔细考虑下还是弄个自己独立的好，毕竟我也是个爱折腾的人。
起初比较二，居然想着自己前端后端写一套，而且还说做就做，前端没有美工，我尼玛居然想到用bootstrap（唉，真是蠢），仍然是怎么搞怎么觉得丑，以前只管切美工做好的图，还真不知道设计页面这么难。然后搞了不少插件，本就不太会写js的我，硬是活生生的给弄的差不多了，虽然坑了一个晚上。
然后就是博客的后台，毕竟php，yii框架相比于前端还是要熟练些的，后台不到一个小时就写完了。
正当我准备买阿里云，把网站挂上去的时候，我偶然间看到了hexo，这个自动生成静态页面的博客框架，我的天，简直改变了我的三观，然后我才意识到之前我为博客做的工作简直就是傻逼，还天真的准备买阿里云，还好及时发现了hexo。

BB了这么多废话，来点干货，总结一下我这些天看的hexo的文档。

hexo是什么？

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

hexo的优点

  • hexo根据你编写的markdown文件，自动将其渲染成静态页面，由于是静态的页面，没有动态数据，不涉及后台以及数据库，因此可以很方便的直接放到github pages， SAE 等平台上。
  • 使用非常的简单，如果你要求不高，那么只需要知道几个简单的命令就可以快速搭建看起来还不错的个人博客，即使你喜欢折腾，也多花不了多久。
  • 有很多的主题供你选择，你无需写任何html，css代码，因为这些都是在你选定博客主题后hexo帮你自动生成的，你只需专注于每篇文章的内容。 网络有很多优雅耐看、功能完善的hexo博客主题，从github上clone一下就好。 如果你能力够强，完全也可以自己编写一个主题。
  • 台湾人写的，理论上中国人用起来还是比较亲切的，而且至少不用担心对中文的支持不够好。
  • 逼格非常之高有木有？反正我觉得是。

开始使用hexo

安装准备

  1. 安装node.js （我不懂nodejs，但貌似hexo是基于nodejs的）
    到node.js官网下载最新版，windows一路next就行。
  2. 安装git （毕竟以后hexo生成的页面是要挂到github上用，部署的时候用git）
    git的客户端很多，我用的是git bash,安装就行。
  3. 安装一个文本编辑器（配置hexo，以及写博客里的文章的时候你总得有个文本编辑器来
    写markdown吧，总不能用word吧！）
    我从大一开始写代码一直用的sublime text, 很轻量级，快捷键方便，而且有很多强大的插件，定制你想要的功能于一体。
  4. 注册github账号，因为到时候需要 将博客页面挂在github pages上运行。（即使不是
    为了搭建这个博客，github这个神奇的代码托管平台，难道不值得一用吗？）
  5. 新建一个github pages仓库。（登陆后，点New repository
    ，填好相关描述，点create。注意github pages仓库和普通的代码仓库略有区别，仓库名必须为『your_user_name.github.com』，也就是说与你的用户名对应。）不出意外的话，your_user_name.github.com 以及可以访问了。
  6. 将你的SSH公钥添加到github上.（告诉github这台设备有权限向你的项目里提交代码）
    具体方法为：
    打开git bash
    设置你这台设备的全局用户名和邮箱

生成SSH密钥 , 输完一路yes和回车就行，默认 密码直接给空。

不出意外的话，这时在电脑的c盘/用户文件夹里多了一个.ssh文件夹，里面有两个文件id_rsa和id_rsa.pub 。 将id_rsa.pub文件里的内容复制到你github的 『Account settings -> SSH Keys -> Add SSH Key』里。

然后可以验证一下是否设置成功，

如果有问题可以参考

  • Github-help Generating SSH keys
  • Github-help Error: Permission denied (publickey)

配置

直到现在hexo博客虽然已经可以正常使用了，但是其中的很多设置都还是hexo默认的，比如说博客的名称，每页显示多少条文章标题等等。 那么这么多跟博客相关的设置在哪里改成自己想要的呢？
在主目录下有一个 _config.yml 文件，这便是配置文件。
用文本编辑器打开它，开始改里面的配置，改完之后保存，执行hexo g 重新渲染一次才生效。
_config.yml里的配置项都是比较简单的英文，而且有注释，分别代表什么功能很容易理解
具体说明参照hexo文档-配置 (有可能需要翻墙才能看)
不能翻墙的话就去网上搜 hexo配置项 ，google用不了百度也行。
